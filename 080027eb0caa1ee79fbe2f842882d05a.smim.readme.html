SAP NetWeaver 7.5 SPS2 - TestDrive

This download provides the software required to install and run SAP NetWeaver Application Server ABAP 7.5 SPS2 (NW7.5) You can explore SAP's current generation ABAP Application server; learn how to develop web applications; and get an overview of SAP's client/server technology.

Please bear in mind that a productive SAP System is designed and optimized for mission-critical applications in a business environment with thousands of parallel users.

Consult the provided SAP DEVELOPER CENTER MASTER SOFTWARE DEVELOPER LICENSE AGREEMENT for SAP-granted usage rights on the included components as listed below.

  • SAP NetWeaver 7.5 SPS2
  • SAP GUI for the Java Environment 7.40 and SAP GUI for Windows 7.40 in recent versions
  • SAP Sybase ASE 16.0.1

Requirements

You need to read and accept the Software End User License Agreement

The 7.5 TestDrive has been successfully tested with recent versions of the following distributions:

  • SUSE SLES
  • Open SUSE
  • Fedora
  • RHEL
  • Ubuntu and Debian: You need to install csh before you start the installation!
  • Other distributions may require manual setup.

It is mandatory that you use a 64-Bit OS version.

Hardware requirements:

  • x86_64 Processor based hardware
  • At least 4 GB RAM plus about 8 GB swap space
  • About 80 GB free disk space for server installation
  • About 2 GB free disk space for client installation

Installation Instructions and First Steps

Step 1: Prerequisites

  • Network configuration of the SAP host operating system: The officially supported setup requires a static IP address for the hostname used for the SAP installation. Using DHCP is possible for this Testdrive version, but be aware of potential consequences. If our installation cannot ping the hostname specified when installing, installation and startup of the SAP system will fail. Make sure that the hostname specified during installation fulfills SAP requirements, most importantly it cannot exceed 13 characters length and also needs to survive subsequent reboots. We strongly recommend that you stick to vhcalnplci as hostname used by the SAP system and make sure that this hostname remains pingable after subsequent restarts of the server.
  • The firewall should be deactivated.

Step 2: Server Installation

  • Login with your preferred user. You need to have the password of the root user at hand, since the installation needs root privileges.
  • Extract downloaded archive: unzip archive.zip.
  • Go into the root directory of the installation, set install.sh to executable and execute:
    1. chmod +x install.sh
    2. sudo ./install.sh
    More detailed installation instructions are provided in dedicated SCN blogs so that we can address your incoming suggestions.
  • Installation will take a while (about 20 minutes - take a coffee).
    1. Start the SAP system
      1. Switch to user npladm with default password Appl1ance in the console : su npladm
      2. Start the SAP system : startsap ALL
    2. Log on to the system as user SAP* with default password Appl1ance and use transaction SLICENSE (transaction code slicense) to ascertain your hardware key. Alternatively, you can get your hardware key by executing /usr/sap/NPL/SYS/exe/run/saplicense -get.
    3. Request the license key for your TestDrive by selecting the value 'NPL - SAP Netweaver 7.x (Sybase ASE)' for System ID and by entering your hardware key in the section System Info on SAP Sneak Preview License Key Request . You should receive an e-mail shortly after sending the data with a license file attached.
    4. In SLICENSE, install the license file.
    5. IMPORTANT: If you log on with the user DEVELOPER, then the first time you create an object, you will be asked for a user key. Enter the following key exactly: 35408798513176413512.
      Please note that all the above steps must be carried out; otherwise, the above user key will not work.

Administration of SAP NW ABAP 7.5 TestDrive

Administration of SAP NW ABAP 7.5 using SAP Management Console SAPMC

    Assuming that your browser is configured to run Java applets and that the NPL server host (example: vhcalnplci) is reachable from your OS, enter http://vhcalnplci:50013. SAPMC allows you to start, stop, and analyse the SAP system and its components with great ease of use, remote and/or local. Alternatively, use the command line sapcontrol commands (not described here) or the deprecated startsap and stopsap commands.

Created directories and a new hostname

  • The installed system creates the following directories:
    • Table 2: Directories
    Directory Size
    /sapmnt ~ 2 GB
    /sybase ~ 50 GB
    /usr/sap ~ 3 GB
  • If you prefer to provide dedicated disk space for the directories shown in table 2, do so before you invoke the script. Please be aware that the used extraction will overwrite dynamic links, so we suggest to use bind mounts in order to persist SAP paths in your host OS /etc/fstab. Assuming that you have made disk space available on directory /sap, /etc/fstab entry /sap/sybase /sybase none bind 0 0 will make the sybase directory (directories must exist!) available at the right location while occupying disk space on the device you have added for running the TestDrive.
  • The installation does not cater for any hostname requirements, so please make sure that you have understood the requirements before you execute the installation script.

Uninstallation Instructions

If you ever want to uninstall your server you can do this as follows:

  1. Delete the created directories shown in table 2,
  2. Delete the users shown in table 3,
  3. Delete the added lines in the file /etc/services
    (should be the last lines beginning with sap* and sql6 * respectively),
  4. Delete the line containing vhcalnplci in the file /etc/hosts,
  5. If applicable to your Linux version: delete the symbolic link S99_vhcalnplci.sh in the directory /etc/init.d/rc3.d in case of SuSE or /etc/rc3.d in case of Red Hat.
  6. Now restart your network.
